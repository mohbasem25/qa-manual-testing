# Security Test Cases — NexaPay Payment Platform

**Platform under test:** the (fictional) NexaPay payment gateway API and Merchant Portal.

These are **functional security checks a QA engineer runs as part of regular test execution** — verifying that documented, publicly known security expectations (aligned in spirit with **PCI-DSS** data-handling principles and the **OWASP API Security Top 10**) hold up in practice. This suite is **not** a substitute for a full penetration test, formal PCI-DSS assessment, or a dedicated security team's threat-modeling exercise; it covers the checks a manual QA engineer is well positioned to perform during functional test cycles.

## Notes

This suite intentionally stays at the level of functional, black-box security checks appropriate for a manual QA engineer's regular test cycles. It references PCI-DSS and the OWASP API Security Top 10 only as general, publicly available frameworks for categorizing risk areas — it makes no claim that NexaPay (a fictional platform) holds any compliance certification, and it is not a replacement for formal penetration testing or a compliance audit.
